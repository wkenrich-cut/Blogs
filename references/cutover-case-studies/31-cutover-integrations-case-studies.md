# Cutover Integrations Case Studies

**Source:** https://cutover.com/success-stories/cutover-integrations-case-studies

---

## Overview

This case study covers multiple real-world examples of how organizations use Cutover's integration capabilities with tools like ServiceNow, Ansible, BMC Remedy, ITRS Geneos, and internal systems to automate manual tasks and improve release and resilience activities.

---

## Introduction

Working with tools that are not closely connected can lead to siloed information and poor visibility. Cutover integrations allow clients to extend the value of their existing tools, bring all organizational data onto one platform, automate manual tasks, and gain enterprise-wide visibility.

---

## ServiceNow and CMDB Integrations

A **British multinational investment bank** was using ServiceNow as its Configuration Management Database (CMDB) to store service and application data, while also using Cutover for resilience activities. The bank needed to bring in data from ServiceNow to augment Cutover runbooks — such as service name, criticality categorization, and core information.

The Cutover-ServiceNow integration creates a regular data feed into Cutover, allowing users to filter and drill down into ServiceNow information without leaving their runbook. This integration provided value from day one and continues to create increasing benefits.

An **American multinational investment bank** had a similar need, integrating its internally-built CMDB with Cutover. Admins only have to update the central data source, and it is automatically made available to Cutover users.

---

## Integrations for Enterprise Resilience Testing

A **British mutual financial institution** needed to improve its resilience testing, as the existing process required people to be involved at every step. They integrated a host of tools with Cutover including **Ansible and Interlink**, with plans to add ServiceNow and Microsoft Teams. Using these integrations reduces risk, eliminates human error, and makes the overall process much quicker.

---

## BMC Remedy Integration for Change Events

An **American multinational investment bank** was using BMC Remedy as its ITIL change management system of record while planning changes in outside systems. The manual process involved creating change tickets in Remedy, then manually updating them during events in Cutover.

Some events had up to **50 different change tickets**, creating an enormous administrative burden. Since the bank was already using Cutover to orchestrate change events, they integrated Remedy with Cutover so change coordinators can query, push, and close Remedy tickets directly through the Cutover interface — ensuring tickets are updated in real time and never missed.

---

## Automation as a Service and Internal Planning Tool Integrations

An **American multinational investment bank** integrated its Automation as a Service (AaaS) platform with Cutover, allowing users to build automated tasks into Cutover runbooks. AaaS tasks auto-start and finish based on automation results, and can be re-fired as many times as required directly from Cutover.

The same bank also integrated Cutover with its internal data center migration planning tool. The integration syncs data in real time between the planning tool and Cutover's event roadmap. Management and application teams can visualize the plan in real time and track progress each weekend on a single dashboard. Cutover runbook completion data is passed back to the golden source planning tool for reporting.

---

## Key Integration Benefits Across All Case Studies

- Eliminated manual data maintenance between systems
- Reduced risk and human error through automation
- Real-time data feeds between source systems and Cutover
- Automated change ticket management (zero missed closures)
- Combined human and automated tasks in a single orchestrator
